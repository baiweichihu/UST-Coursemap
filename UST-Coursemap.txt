HKUST COURSE MAP - 项目说明书
================================

一、项目概述
------------

项目名称：
  USTCourseMap（科大课程地图）

项目背景：
  作为HKUST学生，选课时面临两个核心需求：
  1. 课程安排信息：课程有哪些先修课(prerequisite)、共同先修课(corequisite)、
     排斥课(exclusion)，这些关系复杂且难以直观理解
  2. 课程质量评价：ust.space上有学生对课程的四维度评分(overall, teaching, 
     workload, grading)，但需要分别查询
  
  目前这两个信息源是分离的，学生需要在课程安排网站和ust.space之间来回切换，
  效率低且无法形成整体认知。

项目目标：
  打造一个整合工具，将官方课程安排与真实学生评价结合，并用可视化图形直观展示
  课程之间的依赖关系，帮助学生更科学地规划学业。


二、核心功能模块
----------------

2.1 数据爬取模块

  目标：从两个数据源获取并整合信息

  A. 课程安排网站 (https://w5.ab.ust.hk/wcq/cgi-bin/)
     - URL格式：xxyy（xx=年份，yy=学期：10=Fall, 20=Winter, 30=Spring, 40=Summer）
     - 示例：https://w5.ab.ust.hk/wcq/cgi-bin/2530/（25-26 Spring）
     - 需要提取的数据：
       * 课程代码（如 COMP1021）
       * 课程名称
       * 学分数
       * 先修课文本（pre-req）
       * 共同先修课文本（co-req）  
       * 排斥课文本（exclusion）

  B. 课程评价网站 (https://ust.space/review)
     - 需要模拟登录才能访问
     - 需要提取的数据（每个课程四个维度）：
       * 综合评分 (overall)
       * 教学质量评分 (teaching)
       * 工作量评分 (workload)
       * 给分友好度评分 (grading)

2.2 数据处理与存储模块

  目标：清洗、整合数据，并以结构化方式存储

  A. 先修关系解析
     - 将文本形式的先修要求（如 "(COMP2011 OR COMP2012) AND (MATH1013 OR MATH1014)"）
       解析成表达式树(expression tree)结构
     - 支持与、或、括号等逻辑组合

  B. 数据模型设计
     {
       "course_code": "COMP 1021",
       "title": "Introduction to Computer Science",
       "credits": 3,
       "description": "...",
       "preReqs": {表达式树},
       "coReqs": {表达式树},
       "exclusions": ["COMP 1022", "COMP 1022P"],
       "reviews": {
         "overall": 4.5,
         "teaching": 4.2,
         "workload": 3.8,
         "grading": 4.0
       }
     }

  C. 数据存储格式
     - 使用CSV或JSON格式存储（初期简单，后期可考虑SQLite）

2.3 可视化与图形界面模块

  目标：用图形直观展示课程关系，并整合评分信息

  A. 图结构构建（使用NetworkX）
     - 节点：每门课程
     - 有向边：表示先修关系（A -> B 表示A是B的先修课）
     - 特殊边：用不同样式表示排斥关系

  B. 交互式可视化（使用ipysigma）
     - 节点颜色：可按学院/学系区分
     - 鼠标悬停：显示课程代码和ust.space评分
     - 点击节点：高亮显示该课程的所有上下游关系
     - 搜索功能：快速定位特定课程

  C. 增强功能
     - 筛选器：可选择只显示先修关系/排斥关系
     - 标记功能：允许学生标记已修课程
     - 导出功能：可将当前视图导出为HTML分享


三、技术栈建议
--------------

开发语言：Python（AI训练数据最充足，vibe coding友好）

核心库：
  - 爬虫：requests + beautifulsoup4
  - 模拟登录：selenium 或 playwright
  - 数据处理：pandas
  - 图结构：networkx
  - 可视化：ipysigma
  - 网页部署：streamlit（推荐，最快上线）

开发工具：
  - IDE：Cursor 或 VS Code + GitHub Copilot（AI编程最佳体验）


四、与现有项目(CurriculumMap)的关系
------------------------------------

借鉴点（思想层面）：
  - 用箭头表示先修关系的可视化思路
  - 用红叉/特殊标记表示排斥关系的设计
  - 右键标记已修课程的交互概念

差异化优势（本项目独有）：
  - ✅ 整合ust.space四维评分（CurriculumMap没有）
  - ✅ Web交互式可视化（可导出HTML分享）
  - ✅ Python技术栈（更易维护和扩展）
  - ✅ 网页版可直接访问，无需安装

五、项目价值与意义
------------------

对学生用户：
  - 节省时间：一站式获取课程安排+评价信息
  - 辅助决策：直观看到课程依赖关系，避免选课冲突
  - 规划学业：帮助理解课程体系结构，优化四年学习路径

对开发者（你本人）：
  - 技术成长：爬虫、数据处理、可视化、交互设计全栈实践
  - 作品价值：可放在GitHub作为求职作品，真实解决同学痛点
  - 社区贡献：可能成为科大学生的实用工具，获得认可


六、开发路线图（建议）
----------------------

第一阶段：爬虫开发（1-2天）
  - 实现课程网站爬虫，获取基础数据
  - 保存为CSV/JSON

第二阶段：评分整合（1天）
  - 实现ust.space模拟登录
  - 抓取四维评分，合并到数据集中

第三阶段：可视化原型（2-3天）
  - 用networkx构建图
  - 用ipysigma实现基础可视化

第四阶段：交互增强（1-2天）
  - 添加搜索、筛选、标记功能
  - 优化UI/UX

第五阶段：部署上线（1天）
  - 用Streamlit封装成网页应用
  - 部署到Streamlit Cloud（免费）
  - 可选：再用PyInstaller打包exe供本地使用
