HKUST COURSE MAP - PROJECT BRIEF

1. Overview
USTCourseMap integrates HKUST course dependency information with USTSpace review metrics.
The app helps students understand prerequisite chains, co-requisite links, and exclusions in a graph view.

2. Data Sources
- WCQ course schedule pages: https://w5.ab.ust.hk/wcq/cgi-bin/{semester}/
- USTSpace reviews: https://ust.space/review

3. Data Pipeline
- Crawl WCQ snapshot by semester.
- Crawl USTSpace review snapshot by semester (with authenticated session).
- Merge snapshots into a unified course dataset.
- Build canonical course model and graph payload.

4. Graph Semantics
- Node: one course.
- Edge types:
  - pre_req
  - co_req
  - exclusion
- Mutual exclusion is represented as two opposite exclusion edges.

5. UI Behavior
- Default view is subject overview (e.g., COMP (count)).
- Click a subject to drill down to course-level graph.
- Course-level graph uses radial importance layout (high-importance courses near center).
- Same stem variants (e.g., COMP 4971A/B/C) are grouped to reduce horizontal spread.
- Search supports course code/title.
- Completed course list supports add/remove.
- Current view can be exported as HTML.
- Details panel shows ratings and review count when review count is available in snapshot metadata.
- Double-click external URL jump in upstream streamlit-agraph is disabled in local runtime patch.

5.1 Semester Display Modes
- Supports regular semester entries by snapshot folder.
- Includes virtual options for 2025-2026 scope:
  - 2025-2026 All
  - 2025-2026 Fall (F/S)
- Fall (F/S) mode includes courses appearing in Fall OR Spring (not exclusive-only).

6. Snapshot Policy
- Use one snapshot set per semester.
- Re-crawl only when needed or when force refresh is requested.
- During normal app usage, read local snapshots only.

7. Deployment
- Streamlit app entry: app.py
- Dependencies: requirements.txt
- Optional secrets for crawler actions:
  - USTSPACE_USERNAME
  - USTSPACE_PASSWORD
- Local development can use `.env` + PowerShell loader to populate env vars before crawl commands.
